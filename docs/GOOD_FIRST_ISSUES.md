# Good first issues

No scoped good-first tasks are currently tracked here.

When adding new starter work, keep each task limited and self-contained, with:

- the file(s) to touch,
- the expected outcome,
- the trust boundary when security-sensitive, and
- the local check (`just check` should cover it).

See `CONTRIBUTING.md` for the standard inspect → edit → verify flow and
`ISSUES.md` for current audit findings.
